# Future Feature: Dynamic AI Tutor Content

**Status**: Planned · Not Started

## Summary
Currently, all AI tutor content (component overviews, code line explanations, hints) is hardcoded in Kotlin source files. This feature enables these to be dynamically generated, edited, and evolved.

## Key Capabilities
1. **LLM-Generated Content**: Use the configured LLM to generate explanations on-demand
2. **Content Caching**: Cache generated content locally for offline use
3. **User Editing**: Users can modify explanations/hints to their preference
4. **Content Evolution**: Version-track changes, revert to defaults
5. **Community Sharing**: Export/import custom content packs

## Implementation Ideas
- Store content in Room DB instead of hardcoded files
- Content versioning with timestamps
- Bilingual content can be generated by LLM or user-provided
- Optional: sync content from a community repository

## Dependencies
- Real LLM provider (in progress)
- Room DB schema for content storage
- UI for content editing

## Open Questions
- How to handle content updates without breaking existing progress?
- Should users be able to revert to "official" content?
- How to support multiple content sources/languages?
