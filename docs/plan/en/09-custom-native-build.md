## Issues Encountered & Fixes

### 1. NDK Extraction Corrupted
**Symptom**: `ld.lld` was 3 bytes (symlink to truncated target)  
**Cause**: `unzip` timed out on 507MB extraction over WSL filesystem  
**Fix**: Re-download and extract with `unzip -o` (no piping), increase timeout  
**Status**: ✅ Fixed

### 2. Proxy not mirrored to WSL
**Symptom**: "A localhost proxy configuration was detected but not mirrored into WSL"  
**Cause**: Windows has `http_proxy=http://127.0.0.1:10808` (Clash). WSL NAT can't reach `127.0.0.1`  
**Fix**: Use `http_proxy=http://172.20.80.1:10808` (Windows host IP from WSL)  
**Persist**: Add to `~/.bashrc` in WSL: `export http_proxy=http://172.20.80.1:10808`  
**Status**: ✅ Fixed

### 3. sudo blocks in WSL
**Symptom**: `sudo apt-get install` hangs indefinitely  
**Cause**: sudo requires password, no TTY available from Hermes  
**Fix**: User runs `sudo apt-get install -y libc++1` manually in WSL terminal  
**Status**: ✅ Workaround

### 4. CMake from Android SDK is Windows binary
**Symptom**: `cannot execute binary file`  
**Cause**: `$ANDROID_HOME/cmake/3.22.1/bin/cmake` is a Windows `.exe`, WSL can't run it  
**Fix**: Use WSL-native cmake: `/usr/bin/cmake` (v4.2.1)  
**Status**: ✅ Fixed

### 5. CMake "No SOURCES given to target: llama"
**Symptom**: CMakeLists.txt has sources but CMake says empty  
**Cause**: Multiple `//wsl$/` sync issues — file not reaching WSL  
**Fix**: Write to Windows → `cp` to WSL via terminal  
**Status**: ✅ Fixed

### 6. Missing source files (ggml-backend.c)
**Symptom**: `Cannot find source file: ggml-backend.c`  
**Cause**: llama.cpp restructured — `ggml-backend.c` → `ggml-backend.cpp`  
**Fix**: Updated source list in CMakeLists.txt  
**Status**: ✅ Fixed

### 7. C++17 features without -std=c++17
**Symptom**: `decomposition declarations are a C++17 extension`  
**Cause**: NDK defaults to C++14  
**Fix**: Add `set(CMAKE_CXX_STANDARD 17)`  
**Status**: ✅ Fixed

### 8. Missing GGML_VERSION/GML_COMMIT macros
**Symptom**: `use of undeclared identifier 'GGML_VERSION'`  
**Cause**: These are normally generated by llama.cpp's own CMake configure  
**Fix**: Add `target_compile_definitions(llama PRIVATE GGML_VERSION="undefined")`  
**Status**: ✅ Fixed

### 9. Missing include: ggml-impl.h
**Symptom**: `fatal error: 'ggml-impl.h' file not found`  
**Cause**: New `ggml-cpu` directory not in include path  
**Fix**: Add `${LLAMA_DIR}/ggml/src/ggml-cpu` to include dirs  
**Status**: ✅ Fixed

### 10. Undefined symbols at link time
**Symptom**: 20+ undefined symbols like `ggml_backend_reg_count`, `llama_model_free`  
**Cause**: Our minimal 8-file source list doesn't cover all internal dependencies. llama.cpp has 100+ source files with complex interdependencies.  
**Fix**: Need to build llama.cpp as a full static library using its own CMake, then link our JNI against it.  
**Status**: ⚠️ Current blocker

## Correct Approach (Not Yet Implemented)

### Option A: Build llama.cpp as static lib + link JNI
```bash
cd /tmp/llama.cpp
cmake -B build-android \
  -DCMAKE_TOOLCHAIN_FILE=$NDK/build/cmake/android.toolchain.cmake \
  -DANDROID_ABI=arm64-v8a \
  -DANDROID_PLATFORM=android-26 \
  -DBUILD_SHARED_LIBS=OFF \
  -DGGML_OPENMP=OFF \
  -DLLAMA_CURL=OFF

cmake --build build-android -j8
# → build-android/libllama.a (static library)

# Then build our JNI against this static lib
cd /path/to/our/jni
cmake -B build \
  -DLLAMA_LIB_DIR=/tmp/llama.cpp/build-android \
  -DCMAKE_TOOLCHAIN_FILE=...
# Link: target_link_libraries(llama_jni ${LLAMA_LIB_DIR}/libllama.a)
```

## Current CMakeLists.txt (Working for compilation, failing at link)

```cmake
cmake_minimum_required(VERSION 3.22.1)
project("llama_jni")
set(CMAKE_CXX_STANDARD 17)

set(LLAMA_DIR /tmp/llama.cpp)

add_library(llama SHARED
    llama_jni.cpp
    ${LLAMA_DIR}/src/llama.cpp
    ${LLAMA_DIR}/ggml/src/ggml.c
    ${LLAMA_DIR}/ggml/src/ggml-alloc.c
    ${LLAMA_DIR}/ggml/src/ggml-backend.cpp
    ${LLAMA_DIR}/ggml/src/ggml-quants.c
    ${LLAMA_DIR}/ggml/src/ggml.cpp
    ${LLAMA_DIR}/ggml/src/gguf.cpp
)

target_include_directories(llama PRIVATE
    ${LLAMA_DIR}/include
    ${LLAMA_DIR}/ggml/include
    ${LLAMA_DIR}/ggml/src
    ${LLAMA_DIR}/ggml/src/ggml-cpu
    ${LLAMA_DIR}/src
)

target_compile_definitions(llama PRIVATE GGML_USE_CPU=1)
target_compile_features(llama PRIVATE cxx_std_17)
target_link_libraries(llama android log)
```
