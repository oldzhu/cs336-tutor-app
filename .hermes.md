# CS336 Tutor App — Project Rules

> Auto-injected into every Hermes session when working in this project directory.
> Bilingual docs at docs/spec/, docs/plan/, docs/log/ in CN/EN.

## RULE 1: Test-Driven Development (TDD)

**All features — existing and new — MUST follow TDD:**

1. **RED** — Write failing test FIRST
2. **GREEN** — Minimal code to pass
3. **REFACTOR** — Clean up, keep tests green

### Test Commands
```bash
# Unit tests (Kotlin JVM)
./gradlew testDebug

# Instrumented tests (Android device/emulator)
./gradlew connectedCheck

# Both
./gradlew check
```

### Test Structure
```
app/src/test/        → Kotlin JUnit tests (fast, runs on JVM)
app/src/androidTest/ → Instrumented tests (needs device/emulator)
```

### Before Deploying
- [ ] All unit tests pass
- [ ] Instrumented tests pass on device
- [ ] User manual test confirmation

## RULE 2: OpenCode Session Continuity

**Always resume from the last OpenCode session:**

```bash
# List sessions
opencode session list

# Continue last session
opencode -c

# Continue specific session
opencode -s <session_id>
```

### Session Tracking
- Current session ID is tracked in docs/log/session-id.txt
- OpenCode sessions live in `~/.local/share/opencode/sessions/`
- Each feature gets a named session: `opencode --title "phase-2-rmsnorm"`

## Language & Docs
- All UI strings: English (extractable to strings.xml for i18n)
- All docs: Bilingual CN/EN, cross-linked
- Commit messages: English
- Code comments: English

## RULE 3: Manual Testing Before Moving On

**After automated tests pass, deploy to real device before proceeding:**

1. Auto tests pass → Ask user to connect phone
2. Build APK: `./gradlew assembleDebug`
3. Deploy: `adb install app/build/outputs/apk/debug/app-debug.apk`
4. User manual tests on phone
5. User confirms → Proceed to next feature

**Do NOT move to the next feature until user confirms manual test pass.**
